Python :  파이썬은 직관적인 문법을 가지고 있다. 파이썬 문법을 배운 개발자가 생각하는 그대로 쭉 써내려 가면 바로 코드가 완성된다. 작성이 매우 간결하다는 점은, 읽기도 쉽다는 장점을 가진다. 배운지 하루 만에 원하는 코딩이 가능할까?
파이썬이라면 가능하다. 그만큼 쉬운 문법을 가고 있으며, 문법의 패턴이 사람이 생각하는 패턴과 비슷하기에 구성하기도 쉽다. 다른 언어의 기능들을 모듈화시켜 파이썬에 도입하는 것도 가능하다. 보통 C + 파이썬 조합을 이용한다. 오픈소스 기반이기에 무료로 제공되며, 이는 비용 절감을 의미한다.
직관적이고, 쉽고 간결하다로 요약되는 파이썬이기에, 개발 속도를 획기적으로 빠르게 했다는 점 또한 파이썬의 큰 장점으로 다가온다.

 

Ruby : 루비의 장점 5가지는 다음과 같다. 언어의 모든 것이 객체로 되어 있다는 점이다. 루비는 모든 부분이 자신의 속성과 액션을 부여 받아서 이 것들이 모두 일관되게 적용 받는다는 점에서 루비 언어를 좀 더 쉽게 다루게 해준다.
두 번째는 유연함이다. 루비는 개발자들에게 제한을 두지 않는다. 핵심 부분을 개발자가 새로 만들 수도 있고, 기 코드에 원하는 생각을 마음껏 담을 수 있다.
세 번째는 블록 기능이다. 메서드의 동작 중 일부를 개발자의 블럭으로 채울 수 있는 오픈 구조를 채택함으로써 개발의 자유도를 높인 것 또한 장점이다.
네 번째로 루비는 모듈 개념을 도입하여 클래스에서 모듈에 있는 메서드를 마음껏 얻어 쓸 수 있다는 장점을 가진다.
마지막으로 루비 언어는 네이밍 특징을 줌으로써 가독성, 작성법을 간단하게 했다는 점도 장점으로 가진다.

 

PHP : PHP는 HTML의 틀에서 벗어나 동적인 페이징 처리를 가능하게 해준다는 점이 첫 번째 장점이다. 변수의 바인딩을 통해서 동적으로 받아오는 변수 처리로 인해, 개발자는 손쉬운 개발의 이점을 얻는다.
둘째, 루틴 작업을 쉽게 구현 가능하다. C기반의 웹 프로그래밍 언어이기 때문에 LOOP문의 구현 또한 가능하다는 점은 단순 HTML 대비 더 효율적인 연산 처리를 가능하게 한다.
PHP의 세번째 장점은 개발비용과 개발 속도이다. PHP는 전적으로 무료이기 때문에 환경을 구축하는 데 있어서 큰 비용이 들지 않는다.
그리고 다양한 데이터베이스를 간결한 문장으로 쉽게 연결할 수 있는 이점 또한 존재하며, 문법이 간결하다는 점은 개발속도를 빠르게 하는 장점을 가진다.

 

J-vaScript : 자바스크립트는 PHP처럼 상호작용하는 페이징 처리 구현이 가능하다. 자바스크립트의 가장 큰 장점은 어떠한 개발 툴 없이도 HTML에 삽입하는 형태로 처리가 가능하다는 점이다. 언제 어디서나 OS 무관하게 텍스트 편집기와 인터넷 브라우저만 있다면 자바스크립트는 처리 가능하다.
그리고 변수와 문법 개념을 도입하여 HTML만으로는 하지 못하는 동적 페이징을 가능하게 해준다. 그 문법은 매우 간단하며, 문법의 컴파일 과정 또한 없다.
쉬운 언어에 속하기 때문에 코드의 가독성 또한 좋은 편이다. 뛰어난 기술을 요구하지 않아 간단한 개발부터 도입하기에 매우 적합하다.

 

비교점 : 네 언어 모두 문법이 쉽다. 사용자들이 배우기 쉬운 요소들을 담고 있으며, 개발 속도도 기존 언어들에 비해 빠른 편이며, 개발 비용 또한 들지 않는 특징을 가진 언어들이다.
객체적인 것에만 한정 지어져 있다는 부분(파이썬, 루비)과, 소스 코드 보안 문제 (소스가 그대로 노출될 수 밖에 없는 자바스크립트), 간결화 된 나머지 필요한 것들은 추가적으로 구성해야 하는 점(PHP)은 이 언어들의 단점으로 다가오기도 한다.

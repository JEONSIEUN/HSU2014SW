1)Python

1. 코드가 매우 직관적이어서 이해도가 빠릅니다.
만약에 모든 프로그래밍을 배우면 가장 기초인 Hello 란 글자를 출력한다고 하면 print "Hello Python" 하면 Hello Python 이 출력됩니다.
또 if 3 in [5,1,3,2]: print ("3이 있습니다") 출력하면 “4가 있습니다,” 가 나옵니다. 이처럼 마치 영어를 읽듯이 만약에 3이 5,1,3,2, 안에있으면 “3이 있습니다“를 출력합니다. 
Python이라는 언어의 코딩부분을 처음봤는데 이 부분이 참 쉽게 와 닿았습니다.

2. 다른 프로그래밍과의 연동이 용이합니다.
Python은 찾아본결과 php, oracle, c++, apach, mysql, java등 여러 가지 프로그램과의 연동이 가능합니다. 

3. 메모리의 자동관리로 쉬운 유지보수
Garbage Collection 기능이 있어서 필요할 때 메모리를 자동 할당해주어 필요에 따라 메모리를 사용합니다. 그러므로 사용자가 특별히 메모리 관리를 할 필요가 없습니다.

4. 수많은 라이브러리가 있습니다.
만약에 text라는 문자 배열에 a라는 문자를 찾으려한다면 일일이 a와 비교를 해서 찾아야겠지만 Python안의 기능을 사용하면은 text.count('a‘) 라는 기능을 불러오면 간단하게 구현해낼 수 있습니다. 이러한 라이브러리 기능이 Python안에 내장 라이브러리나 외부에서 불러와서 구현이 가능하므로 적극사용하고 알아두면 좀더 프로그래밍을 할 때 도움이 됩니다.

5. 프로그램 개발속도가 매우 빠릅니다.
다른 C나Java에 비해 보다 코딩이 간결해서 개발속도가 매우 빠릅니다.



2)Ruby

1. 배우기 쉽습니다.
루비의 개발자 마츠모토 유키히로는 프로그래밍 언어는 인간중심으로 설계가 되야한다고 생각하고 만든 언어 이기 때문에 배우기가 쉽습니다. 
가장 기초적인 코딩을 해보면 puts("Hello, Ruby!")  하면  Hello, Ruby! 가 출력됩니다.
2. 순수 객체지향 언어이다.
루비는 모든 것이 객체이다. x=nill 이라면 nill이란 아무것도 없는 값인데 이것또한 객체가 됩니다.

3. 생산성이 높습니다.
같은 코드를 다른 언어로 짤 때 보다 훨씬 더 직관적이기 때문에 같은 코드를 더 빨리 작성하고 다른 일에 몰두 할 수 있다.

4. 매우 유연한 언어입니다.
만야에 a=1+3 이라는 코딩을 만들다 +가 보기 싫다 plus라고 보고싶다하면 a=1 plus 3으로 바꿔줄수도있을 정도로 자유도가 높습니다.
 
5. 이식성이 높습니다.
리눅스, 윈도우, 도스, 유닉스, 맥 등 대부분의 OS 에서 지원합니다.






*)Ruby 와 Phython 비교
Ruby가 좀더 객체지향적이고 좀더 프로그램 문법상에 있어서도 자유롭습니다. 또 웹에플리케이션 개발에 있어서도 보다 도움을 줍니다. 또 기존 다른 프로그래밍 언어를 익힌사람은 Ruby보단 Phython이 이해가 더 쉽습니다. 그리고 세계적으로 Ruby보다 Phython 이용자가 더 많습니다. 



3)PHP

1. 개발 속도가 빠릅니다.

2. 데이터베이스와 연동이 쉽습니다.
oracle, sybase, msql, mysql 등 대부분의 데이터 베이스와 연동이 됩니다.

3. 호환성이 좋습니다.
대부분의 운영체제에서 모두 구현이 가능합니다.

4. 웹개발에 특화된 언어입니다.
처음 개발부터 개인 홈페이지를 관리하기위해 만들어졌기 때문에 웹에 특화되어있습니다.

5. 처리속도가 빠릅니다.
다른 언어보다 변환하고 해석하는 과정을 덜거치기 때문에 상대적으로 더 빠릅니다.


4)Javascript

1. 별도의 프로그램 없이 사용이 가능합니다.
메모장이나 기타 편집 프로그램에 저장하고 바로 테스트가 가능합니다.

2. 처리 속도가 빠르다.
브라우져에서 바로 실행됩니다. 그래서 다른 프로그램에 영향 받지않고 빠르게 실행됩니다.

3. 애니메이션 효과를 만들 수 있다.
기존에 텍스트만 있던 인터넷 환경에서 이벤트를 이용해 마우스로 그림을 움직일 수도 있고 글에 효과를 주는등 다양한 효과를 줄 수 있습니다. 

4. 시스템에 부담이 적다.
다른 언어와 비교하여 간결하고 HTML 안에 삽입된 형태이기 별도 네트워크 연길이 필요없기 때문에 시스템에 부담이 적습니다.
5. 호환성이 좋아 운영체제에 제한을 받지 않는다.
HTML 소스 안에서 전송 되기 때문에 운동체제 제한없이 작동 합니다.







*)PHP 와 Javascript 비교
좀 더 소규모의 프로젝트를 진행하면 PHP가 좋습니다. 개발환경구축이나 서버환경구축등 PHP가 개인홈페이지 관리를 바탕으로 발전해온 언어이기 때문에 보다 적합니다. 
대규모 프로젝트를 하면 Java가 보다 좋습니다. 기능적인면 보단 유지보수 적인 측면에서 보다 더 적합하기 때문입니다.
